## 不可违反的创作边界

- 不建立修炼、法力、等级、超能力或可验证的妖鬼谱系。
- “执念”是创作方法与人物解释，不是作者确认的世界真相。
- 每个怪谈至少保留一种合理解释，作者不宣布最终答案。
- 不为了反转而反转，不使用现代网络梗。
- 不把真实历史人物工具化，不改变已发生的大历史。
- 世界大于主角，人物大于设定，情感大于情节机关。

## 史料与原创

- 事实性内容必须标注A/B/C/D/待考等级，定义见 `SOURCES.md`。
- 无可靠来源的历史、民俗、物价、职业规矩和黑话默认【待考】。

- 资料之间冲突时保留出处和分歧，不擅自拼成唯一版本。

## 文档维护规则

- 00—19号文件是现行核心文档，编号连续；新增或改名必须同步更新 README 和所有引用。
- 修改设定文档时更新头部 Version、更新时间，并在 `CHANGELOG.md` 记录重要变化。
- 创作决策进入 `创作日志.md`；一次性操作过程留在 Git 历史，不塞进本文件。
- README 面向新人，只写当前事实和使用方法，不写会话流水账。
- 不建立与根目录编号文档重复的 `docs/` 副本。
