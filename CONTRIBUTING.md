# 贡献指南

感谢你对山河异闻计划的关注。

本项目是一个开放的世界观设定库。欢迎通过 Issue 或 PR 贡献内容。

---

## 可以贡献什么

### 世界观设定

- 新的职业设定
- 新的地域风土描述
- 新的民俗文化细节
- 新的民间俗语、黑话、歇后语
- 历史事件补充

### 故事素材

- 新的怪谈种子
- 新的人物设定
- 新的伏笔线索
- 经典台词

### 错误修复

- 时间线冲突
- 设定矛盾
- 历史事实错误
- 文字错误

---

## 贡献流程

### 通过 Issue

1. 使用对应的 Issue 模板
2. 填写必要信息
3. 等待讨论和确认

### 通过 PR

1. Fork 本仓库
2. 创建分支：`git checkout -b feat/你的设定名称`
3. 修改文件，更新文件头部的 Version
4. 更新 `CHANGELOG.md`
5. 提交 PR，使用 PR 模板

---

## 提交规范

使用以下前缀：

| 前缀 | 含义 |
|------|------|
| `feat:` | 新设定/新内容 |
| `fix:` | 修复冲突/错误 |
| `docs:` | 文档更新 |
| `style:` | 格式调整 |
| `refactor:` | 重构设定 |

示例：
```
feat: 新增扬州怪谈
fix: 修改时间线冲突（万历十五年）
docs: 更新民俗设定
```

---

## 原则

在贡献内容之前，请阅读以下文件：

1. **`宪法.md`** — 十二条创作宪法（最高准则）
2. **`00_创作理念.md`** — 创作纲领

所有贡献必须符合这两份文件的精神。

### 特别注意

- 不要给出怪谈的"官方解释"
- 不要使用现代网络用语
- 历史人物不得与真实历史矛盾
- 每个设定至少保留两种解释的可能性
- 新增文件请使用 Version 标注

---

## 文件规范

### 文件命名

- 世界观设定：`XX_中文名称.md`（两位数编号）
- 100+ 编号：核心资产文件
- 人物文件（未来）：`characters/人名.md`
- 故事文件（未来）：`stories/第X卷/章节.md`

### 文件头

每个文件必须包含 YAML frontmatter：

```markdown
---
Version: 0.1
更新时间: YYYY/M/D
状态: 简要描述
---
```

### 单文件长度

建议 2000—5000 字。超过 5000 字的设定建议拆分。

## 讨论

- 对世界观有疑问或建议，使用 Issue 讨论
- 对现有设定有不同看法，欢迎在 Issue 中提出
- 所有重大修改需要先开 Issue 讨论，确认后再提交 PR
