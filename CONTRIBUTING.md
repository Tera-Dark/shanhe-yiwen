# 贡献指南

> “山河异闻计划 / Project ShanHe”是开发代号，正式作品名待定。

## 可以贡献什么

- 历史资料校正；
- 职业、地域、民俗与组织资料；
- 怪谈种子；
- 人物与故事提案；
- 时间线、称谓和语言纠错；
- 文档结构与项目管理改进。

## 提交前必读

1. `宪法.md`
2. `00_创作理念.md`
3. `README.md`
4. `SOURCES.md`
5. 与本次修改相关的设定文件

## 史料与原创必须分开

提交事实性材料时，请标注史料等级、年代与地域、来源信息、推断部分和项目原创部分。禁止把后世民俗、现代旅游名片、武侠惯例或网络说法直接写成明代事实。

## 贡献流程

### 公开贡献流程

1. 使用 Issue 记录人物、故事、设定或事实修复；
2. 外部贡献者 Fork 仓库，从最新 `main` 创建功能分支；有写权限的协作者可直接创建分支；
3. 修改相关文件并更新设定文件 Version；
4. 更新 `CHANGELOG.md`；必要时更新 `创作日志.md` 与 `SOURCES.md`；
5. 提交 Pull Request，使用模板自检后等待审查与合并。

公开仓库不等于放弃许可、史料和质量要求。未经维护者确认，不得修改正式名称、许可范围或仓库治理设置。

## Commit 前缀

| 前缀 | 用途 |
|---|---|
| `feat:` | 新设定或新内容 |
| `fix:` | 修复事实、时间线或设定冲突 |
| `docs:` | 文档与说明更新 |
| `style:` | 不改变含义的格式调整 |
| `refactor:` | 重组既有设定 |
| `research:` | 新增史料、来源或考证 |

## 版本头适用范围

世界观、人物、故事和素材文档应包含项目版本头。README、LICENSE、CHANGELOG、CONTRIBUTING、GitHub模板和标准工具配置文件豁免。

## 许可约定

除非提交者在 PR 中明确声明并与维护者另行达成书面约定，贡献到设定文档中的内容视为同意按照 CC BY-NC-SA 4.0 收录和再分发。具体小说正文、封面和第三方材料不自动适用该许可。
