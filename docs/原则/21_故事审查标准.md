# 故事审查标准

> **20** 管怎么写。**21** 管怎么审。**postcheck** 管机器底线。  
> 本文件 **Version 0.4** 为磁盘唯一法源。`AGENTS` / catalog / 网页宣称的 `standard_version` **必须**与此一致。  
> 本项目是一个**可承载多种故事的世界**，不是单一志异短篇流水线。  
> 审查只守「像这个世界里的人在过日子、在行走、在动心、在出事」；  
> **不**用某一种结尾、某一种怪谈解释、某一种单元剧公式去卡死所有作品。

---

## 〇、世界与故事形态（审查前先认清）

### 世界底色（方向）

- **轻玄幻的武侠底盘**：有江湖事、门派恩怨可写，有刀剑身法与超常意象；**不走**升级打怪数值流。
- **小架空**：以明中后期气质与山川制度为骨架，允许虚构地名与项目原创规矩；人物可把传说当真，作者仍不把神妖异术写成可验证物种谱系。
- **江湖优先**：镖路、仇隙、规矩、饭碗、面子、门派与散人。
- **可适当糅合现代概念**：组织、契约、舆论、身份焦虑、信息差等结构可化进处境；禁止裸奔网络梗、短视频腔、职场 PPT 黑话。
- **形态开放**：日常、情爱、诡异、大事件、多视角诡计、强意象、群像……都可以。

### 每篇故事先填一张「形态卡」（写进 01_设定 或 07 报告头）

| 项 | 例 |
| --- | --- |
| 主类型 | 江湖 / 日常 / 情爱 / 诡异 / 大事件 / 诡计 / 意象志异 / 混合 |
| 视角 | 单一人称 / 多视角 / 限制知 / 不可靠叙述 … |
| 异能与超常 | 无 / 氛围暗示 / 人物坚信或传闻“坐实”（作者仍不确认谱系） |
| 历史锚定 | 严格贴某年某地 / 小架空虚构县 / 年号模糊 … |
| 现代概念融合 | 无 / 有（写明化用了什么结构） |
| 本篇**不**适用的模块 | 例如：本篇无怪谈 → 跳过 M-怪谈 |

审查时：**S0 → 核心 R0–R3 → 形态卡勾选的模块**。未勾选的模块不评分、不否决。

## 〇·五、S0 · 归属审查（先于 R1–R3）

S0 不做分数，做**能否进正文**的四问。四问任一条硬否 → 不得朱注/验讫（通常 `revise` 或退回世界卡）。

### S0-1 人物核

遮住刀剑、怪谈和历史名词后，是否还剩一个具体人的利害（饭碗、面子、情义、规矩、身体、名声）？  
若只剩「某局规矩很有趣」→ 降为世界卡，不算故事正文过关。

### S0-2 泥层

写的是大事本身，还是大事压到某个普通人身上的重量？  
大事件可以写，但必须落到脸、饭碗、关系、身体、名声或选择上。

### S0-3 非替代

武功、怪谈、巧合是否替人物完成了本应由人物承担的选择？  
超常可以改变处境，**不能替人做决定**（与红灯 5、R2 一致）。

### S0-4 余味来源

读完后留下的是人物、画面和问题，还是作者说出的项目理念金句？  
若只记得「泥还在转」而不记得谁失去了什么 → 不通过朱注/验讫（可退修或黄灯强制改）。

主线章另须对照 `docs/原则/25_踏线人主章公约.md`（懂/债/名）。

## 一、何时必须审查

| 触发点 | 最低深度 |
| --- | --- |
| 新故事初稿完成 | S0 + R0–R3 + 形态卡模块 |
| 大段重写 / 换类型 / 换视角 | 同上，完整重做；审印失效 |
| 只改语词、标点、小段对白 | R0；对白动则加抽查声纹 |
| 试读 / 外发前 | S0 + 核心 + 已选模块；写或追加报告 |

**无怪谈的故事**不缴“怪谈税”；但只要出现怪谈或异象，就必须按 M-怪谈三级边界处理。

## 二、双视角（始终保留，但问法随类型变）

### 读者：只读正文

1. 愿不愿意看下去？  
2. 读完脑子里剩什么（人、事、画面、情绪）？  
3. 有没有被作者按头总结？  
4. 这个故事**作为它声称的类型**是否成立？  
5. 是否还想再进这个世界？

### 编辑：正文 + 本篇设定包

1. 与形态卡是否一致？  
2. 人物是否像活人，而非设定说明书？  
3. 超常/武/江湖若出现，是否服务于人与事？  
4. 现代概念是「化进去了」还是「穿帮了」？  
5. 系列时间线、称呼、许可与状态有无硬伤？

## 三、核心底线（每篇必做 · R0–R3）

### R0 · 机器底线

```bash
python3 scripts/postcheck_story.py stories/<地脉组>/<编号_标题>/正文.md
# 外发前
python3 scripts/postcheck_story.py stories/<地脉组>/<编号_标题>/正文.md --strict
```

过 R0 再谈人工。**R0 PASS 绝不自动改变 `review.state`。**

### R1 · 可读与完成度

- 有清晰的阅读承诺并基本兑现。  
- 不是设定散文冒充故事：要有人物选择、冲突或可感变化。  
- 没有大面积未完成括号、大纲残留、前后直接打架的硬伤。

### R2 · 人物与世界气味

- 主要人物有立场、局限、自己的利害。  
- 世界在故事边角仍在转。  
- 小架空与武侠允许；人物说话办事要像活在自家规矩里。

### R3 · 语言与声口（底线版）

- 对白大体有身份差。  
- 禁止：未消化的网络梗、短视频口头禅、现代职场黑话硬套。  
- 允许：把当代可感的处境翻译成江湖/市井/官场语言。

## 四、可选模块（按形态卡勾选）

### M-江湖 / 武戏

- 冲突落到规矩、恩怨、饭碗、位置，而非只有打戏说明书。  
- 武与异术是「事上见」，不是等级表展览（见 `docs/世界/27`）。

### M-怪谈 / 双解（出现怪谈或异象时必启用）

**三级边界（v0.4 澄清，取代「可以有妖为真」与红灯 5 的冲突表述）：**

| 层级 | 允许 | 禁止（作者口径） |
| --- | --- | --- |
| **可确认** | 碗确实空了；某人确实听见了声；旗确实未沾泥；某人会一门武功/奇术手艺；机关、迷香、驭虫等**可学可练**的手法存在 | — |
| **可强烈暗示** | 超常读法更有魅力，理性解释可以勉强，但**至少保留一条**不违反常理的解释路径 | 作者宣布「唯一真相是鬼」 |
| **不可确认** | — | 鬼吃了碗；旗有神力；某类妖是可验证物种；执念是宇宙能量；内功能辨妖；固定妖谱/法力技能树 |

- **必须**：至少一种合理解释路径仍在（读者可选择不信超常）。  
- **作者不得**宣布最终超自然答案，也**不得**建立可验证妖鬼谱系。  
- 武学可确认（师承苦练），异闻不确认——与 `27` 分账一致。

### M-强意象 / M-多视角 / M-情爱 / M-日常 / M-大事件 / M-现代概念融合 / M-系列钩子

（要求同 v0.3：意象有根、诡计基本公平、情爱有阻碍、日常不展览民俗、大事件落到脸上、现代概念要化进去、独立可读优先。）

## 五、红灯（一票否决）

1. R0 阻断级未修。  
2. 全文无法辨认人物利害，纯设定或纯金句展览（含 S0 硬否）。  
3. 歧视、虐待等描写无叙事必要且以猎奇为目的。  
4. 把**未标原创/架空**的内容伪装成不可质疑的史实。  
5. 把修炼数值、法力等级、超能力技能树，或**可验证妖鬼谱系**写成作者确认的世界事实。（单个现象「可确认」见 M-怪谈；**谱系/物种学**仍否。）  
6. **超常替代人物选择**（S0-3）。  
7. 许可与仓库安全红线（见 `AGENTS.md`）。

**下列不是全项目红灯**：结尾必须不治愈；必须官方记录反讽；必须单元剧 1–5 万字。

---

## 六、黄灯（报告记一笔即可）

- 作者腔总结过多；同一章选择后连续解释两次以上。  
- 门楣式金句过量（建议一章最多一句「泥/债/名」式收束）。  
- 相邻章节同构（听—停—不问）无新后果。  
- 档案称呼与正文不一致；现代概念边缘穿帮。  
- 武戏或意象过长压死人。  
- 字量明显低于本卷舒适线（主线厚章目标见卷总纲；短幕间可标注）。

黄灯**不自动**阻止「朱注」；外发前应处理。S1/S2 见 7.3。

## 七、结论、审印与网页状态

| `review.state` | 审印 | 含义 | 是否算过关 |
| --- | --- | --- | --- |
| `passed` | **验讫** | S0+R0–R3 通过；启用模块无必须返修项 | **是** |
| `passed_with_notes` | **朱注** | 核心通过，有明确黄灯或外发前修订项 | **是，附注** |
| `revise` | **退修** | 红灯、S1/S2，或 S0/主章公约未过 | **否** |
| `pending` | **待勘** | 未完成当前版本人工审查，或审印已失效 | **否** |
| `exempt` | **卷宗** | 非故事正文 | 不计 |

### 7.1 “过关”的最低条件（硬规则 · v0.4）

只有**同时**满足，才能写入 `passed` / `passed_with_notes`：

1. 正文 `Version` 与 `review.source_version` **严格一致**；  
2. R0 `--strict` 通过；  
3. **人工**完成 S0 + R1–R3（及启用模块）；  
4. `review.round` **≥ 1**（从 0 起算的新建脚手架不得直接过关）；  
5. `review.date` 合法；`review.reviewer` 非空；  
6. `review.standard` 指向本文件，`standard_version` **等于本文件 Version（当前 `0.4`）**；  
7. `review.report` 路径存在，且报告正文中能检索到本条目 **id**（或总簿明确列出本 id 的结论节）；  
8. `review.note` / `notes` **不得**含：`待人工`、`待写`、`待勘`、`脚手架新建`、`待复审` 等未完成语义（可写「已复审；黄灯：…」）；  
9. 主线章额外通过 `25`；外发/连载额外核对 `24`。

**禁止：**

- 仅因 R0 PASS 或 `status: 基础通过` 自动盖印；  
- `round = 0` 写入 `passed*`；  
- note 写着「待人工复审」却 state 为 `passed*`。

### 7.2 审印失效 → 退回 `pending`

- 正文 Version 变化；大段重写、换视角/类型/结尾；  
- 关联设定重大变化；`track` / `main_path` 导致本章功能变化；  
- 上游因果穿帮；  
- 本标准 **Version** 升级且条目未按新标准复审（可选批量重置）。

只改错别字/标点：可保留审印，报告记「轻改核验」。

### 7.3 问题严重度

| 级别 | 含义 | 结论 |
| --- | --- | --- |
| **S1** | 世界规则、历史红线、主因果或人物根本动机崩坏 | 必须 `revise` |
| **S2** | 明显影响独立性、可信度、连载留存或主副线边界 | 通常 `revise` |
| **S3** | 局部重复、声口趋同、时代词、节奏、字量黄灯 | 可朱注 |
| **S4** | 风格建议 | 不阻止通过 |

### 7.4 catalog 字段示例

```json
{
  "review": {
    "state": "passed_with_notes",
    "seal": "朱注",
    "round": 1,
    "date": "2026-07-23",
    "scope": "base+main",
    "standard": "docs/原则/21_故事审查标准.md",
    "standard_version": "0.4",
    "report": "stories/审查总簿.md",
    "reviewer": "总编审",
    "source_version": "1.2",
    "note": "S0–R3 已过；黄灯：字量偏薄，外发前可再厚。"
  }
}
```

- `status` 管稿件阶段；`review.state` 管审查结论，不得混用。  
- 网页只根据 `review` 盖印；缺字段或校验失败 → 显示「待勘」。  
- `seal` 必须与 `state` 匹配：验讫↔passed，朱注↔passed_with_notes，退修↔revise，待勘↔pending。

## 九、工作流

```text
形态卡 → 写（20 + 设定按需）→ 正文
  → R0 postcheck --strict
  → S0 四问
  → R1–R3 + 启用 M-模块
  → 07 报告或总簿（须含条目 id）
  → 红灯/S1-S2 退修；黄灯可朱注
  → catalog.review（round≥1，standard_version=0.4，note 无「待*」）
  → 网页审印
```

## 十一、快速对照卡

1. 核 Version 是否使旧印失效  
2. postcheck --strict  
3. S0 四问  
4. 读者通读 + 编辑红灯  
5. 主线加查懂/债/名与跨章后果  
6. 写报告（含 id）→ round≥1 → 更新 catalog  

**不要：**

- 用某一旧篇的制度反讽当唯一正确结尾  
- 无怪谈强塞双解；有异象则守三级边界  
- 用 R0 或加肉轮次冒充人工朱注/验讫  
- 审查写得比正文还长
