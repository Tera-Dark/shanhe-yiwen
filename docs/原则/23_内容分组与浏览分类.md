# 内容分组与浏览分类

> 一篇故事可以挂多条标签，但**只进一个主架**。  
> 分类为了逛和写，不为了做游戏技能树。  
> 与 `22_内容体例`（写什么体裁）、`11` 总目（有什么条目）配合使用。

---

## 一、推荐模型：一面主架 + 四面筛子

```text
        ┌──────────── 主架：地脉组（读者默认首页）────────────┐
        │  河东线 · 桥头震后 · （未来）江南雨巷 · 京畿 …      │
        └──────────────────────┬─────────────────────────────┘
                               │
         筛子（可多选叠加，不互斥）
         ├─ 体例  Z/P/C/Q/Y/W/R/G/X/S
         ├─ 气味  江湖 / 市井 / 官场 / 志异 / 情爱 / 大事件 / 意象 …
         ├─ 人物·势力  冷旗、青驿、宋婆 …
         └─ 状态  种子 / 初稿 / 基础通过 / 定稿
```

| 层 | 名称 | 作用 | 一条内容有几个 |
|---|---|---|---|
| **主架** | **地脉组** | 文件夹、系列感、互链核 | **恰好 1** |
| 筛 1 | 体例 | 怎么读、多长、什么形态 | 1（主）+ 可备注 |
| 筛 2 | 气味 | 情绪与类型预期 | 1–3 |
| 筛 3 | 人物/势力 | 找「谁的戏」 | 0–n |
| 筛 4 | 状态 | 编辑用 | 1 |

**不要**做成「先选体例再选地区」的深树——燕云感来自「我在这片地上晃」，不是「我在任务类型里打勾」。

---

## 二、主架：地脉组（最重要）

### 定义

**地脉组** = 共享同一条路 / 同一座城 / 同一场灾 / 同一互链核的一组正文。

例：

| 地脉组 ID | 名称 | 互链核（例子） | 已有 |
|---|---|---|---|
| `hedong` | 河东线 | 半截碑 · 雨旗 · 桥心钱 · 青泥渡 · 青驿 | Y/R/G/Q/C 001 |
| `qiaotou` | 桥头震后 | 第二碗 · 勘灾册 · 宋婆 · 华县地震余波 | S001/P001 |
| （未来） | 江南雨巷 / 京畿厂卫 / 运河夜航 … | 各自钉子 | — |

### 仓库怎么放

```text
stories/
  catalog.json
  河东线/           ← 地脉组目录（主架）
    README.md       ← 组简介 + 互链核 + 条目表
    07_组审查.md
    Y001_…/正文.md
    Q001_…/正文.md
  桥头震后/
    P001_桥上第二碗/
```

规则：

1. **新正文默认进某个地脉组**；组未立则先立组 README（三句话 + 互链核）。  
2. 跨组出场可以，**主档仍只在一组**；另一组 README「另见」。  
3. 正文唯一源在 `stories/<地脉组>/`；`archive/` 只放早期短稿与开工草案，不跟主架抢正文。

### 网页怎么呈现

- 首页 / 第一屏：**地脉图或地脉列表**（河东线、桥头震后…）  
- 点进组：组序言 + 互链核 + 组内条目（可再按体例排序）  
- 这就是「燕云式」的文本版跑图感

## 三、筛子细目

### 1. 体例（已有，沿用 22）

`Z P C Q Y W R G X` + 旧 `S`  

网页侧栏可固定一排图标/印章：主纪、旁纪、奇遇、探游…

### 2. 气味（建议受控词表，少而稳）

| 气味标签 | 读者预期 | 例 |
|---|---|---|
| `江湖` | 刀、规矩、镖路、恩怨 | Q001 G001 |
| `市井` | 饭碗、街巷、灾后日常 | S001 Y001 |
| `官场` | 册、帖、差、衙 | S001 局部 |
| `志异` | 说不清、双读、轻玄 | Q001 C001 |
| `情爱` | 关系与阻碍 | （空） |
| `大事件` | 战、灾、朝变压到人肩 | 华县地震背景 |
| `意象` | 强画面核 | （未来 X） |
| `传闻` | 不可靠、打架说法 | C 贩子；未来 W |

一条 1–3 个；**第一个为气味主标签**。

### 3. 人物 / 势力

从正文 frontmatter 或组 README 抽：

- 人物：冷旗、阿段、柳先生、老成、宋婆、陈七…  
- 势力：青驿镖局、（未来）某门某帮  

网页做「点人出戏」即可，不必先做完整族谱。

### 4. 状态

`种子 | 大纲 | 进行中 | 初稿 | 基础通过 | 定稿`  

编辑视图默认显示；读者视图可只露「可阅 / 撰写中」。

## 四、每条正文建议的元数据（给前端）

在 `正文.md` 头或旁路 `meta.yaml` 中保持稳定字段：

```yaml
id: Q001
title: 旗先过桥
genre: Q                 # 体例
group: hedong            # 地脉组 ID
group_name: 河东线
scent: [江湖, 志异]       # 气味，首项主标签
people: [冷旗, 阿牛]
factions: [青驿镖局]
links: [Y001, R001, G001, C001]
status: 基础通过
era_feel: 明中后期·小架空
path: stories/河东线/Q001_旗先过桥/正文.md
```

前端**只认字段，不认文件夹深度**——文件夹给人类用，字段给机器用。

## 五、网页信息架构（给另一 AI 的推荐）

```text
[山河异闻]
│
├─ 入世（首页）
│   · 一句话世界观
│   · 地脉组卡片（主入口）
│   · 最近更新 / 可阅条目
│
├─ 地脉
│   · 河东线 → 组页（核 + 列表）
│   · 桥头震后 → …
│
├─ 体例（第二入口，适合「想看奇遇/探游」）
│   · 按 Z/P/Q/Y… 浏览
│
├─ 人物·势力（第三入口，可后做）
│
├─ 卷宗（设定 00–23、年表——折叠，不抢故事）
│
└─ 单篇阅读页
    · 正文
    · 形态/气味标签
    · 另见（互链）
    · 同组其它
```

视觉：中式江湖——不必仿游戏 UI 控件；用「卷、帖、路引、茶馆榜」做分区隐喻即可。

## 六、不推荐的分法（踩过的坑）

| 分法 | 问题 |
|---|---|
| 只按体例 | 河东线被拆碎，互链感死 |
| 只按时代年号 | 小架空年号含糊，空组多 |
| 只按「主线/支线」 | 世界不是单线游戏 |
| 深树四级文件夹 | 写手与前端都痛苦 |
| 气味自由标签无词表 | 三个月后不可筛 |

## 八、一句话决策

**给人走的：地脉组。给筛选的：体例 + 气味。给迷的：人物。给编辑的：状态。**

主架稳了，体例怎么加都不炸；主架乱了，标签再多也只是散页。

---

*路是骨架，体例是步法，气味是天气。*  
*先让人上路，再让人选今天走哪一种步法。*
