# 卷册与连载阅读体验

> **23** 定地脉主架。**22** 定体例笔墨。**24** 定「怎么装成一本可读的书」。  
> 仓库仍是地脉资料本；连载平台（番茄等）是**精选路径的阅读本**。同源，不同编排。

---

## 一、一句话

**地脉成卷，卷内有序；体例是标签，不是货架。**

- 书架 / 书名层：按**卷**（≈ 地脉组的发布形态）  
- 目录层：按**荐读序**（番茄追更路径）  
- 工程层：仍按地脉组文件夹 + 体例编号  

禁止：随机碎片当主连载；禁止只贴工程编号当章名。

---

## 二、卷（Volume）定义

| 字段 | 说明 |
|---|---|
| `id` | 如 `vol-hedong` |
| `name` | 读者向书名核，如「河东卷」 |
| `group` | 绑定唯一地脉组 `hedong` |
| `subtitle` | 一句气味，如「雨后的官道」 |
| `reading_path` | 荐读序：entry id 数组（可含入卷篇） |
| `status` | 草稿 / 可发 / 连载中 / 本卷收束 |

规则：

1. **一卷只绑一个地脉组**；一组可对应一卷（初期 1:1）。  
2. 跨卷只「另见」，不在章流里双线跳切。  
3. 卷名可用于番茄书名：`山河异闻·河东卷`（开发代号勿当正式书名定死）。  
4. `stories/catalog.json` 的 `volumes[]` 为机器源；网页地脉页优先展示荐读序。

## 三、番茄（类 App 连载）阅读体验指标

写新稿、扩写、排章时，**对照下列指标**。仓库审查（21）过 R0–R3 之后，发平台前再过本表。

### R-读 · 单章可停

| 指标 | 合格线 | 不合格 |
|---|---|---|
| 单章体感字数 | 约 2000–4500 汉字为舒适区；短于 1500 须标明「幕间」或与邻章捆发 | 连续多章不足千字且无标幕间 |
| 章末钩子 | 有「事未完 / 话未说满 / 下一脚在哪」之一 | 纯设定说明书戛然而止 |
| 章名 | 文学短名；可用「幕间·」「残页·」 | Y001、第一节、工程路径 |

### R-追 · 卷有脊柱

| 指标 | 合格线 | 不合格 |
|---|---|---|
| 开卷三章 | 能回答：我在哪条路上、这卷讲什么气味 | 前三章全是残章/纯设定 |
| 主高峰 | 每卷至少一个完整向 Q 或 P（出事—选择—散） | 只有探游+传闻飘着 |
| 收束 | 允许残、阙、不揭真相；须有「卷可停」感 | 为完结强行唯一真相 |
| 更新预期 | 简介写清：群像地脉、非单人升级 | 误导成无敌流后再翻车 |

### R-界 · 与项目核一致

| 指标 | 合格线 |
|---|---|
| 双读 | 志异处理/奇可并存，作者简介不站队 |
| 世界大于主角 | 人走了，线/渡口/局还在 |
| 禁区 | 无数值修炼；无网络梗；不写满须阙的残章 |

### R-装 · 体例怎么装进连载

| 体例 | 番茄中的角色 | 组装建议 |
|---|---|---|
| P 旁纪 | 主章、拆章连载主力 | 按节拆，章名文学化 |
| Q 奇遇 | 主章或上下两章 | 起遇—局中—散，章末留刺 |
| Y 探游 | 开卷定场或卷中换气 | 可单独一章；勿连发三章纯走路 |
| W 传闻 | **幕间** | 短则捆发或标「幕间·」 |
| C 残章 | **残页/附录感** | 靠后或奇遇之后；标题标明残 |
| R/G | 勿连续说明书 | 改写成场景志，或拆进 Q/P 边角；必要时上下两章「志」 |
| V 入卷 | **卷首说书** | 见下节「入卷声口」；非说明书、非作者寄语 |

## 三附、入卷声口（说书人）

每卷开卷一篇（genre `V` / role `prologue`）用**娓娓说书**带读者进地脉，不写工程目录。

| 要 | 不要 |
|---|---|
| 「诸位且坐」式的低声引路 | 「本卷结构如下」「track 说明」 |
| 气味、忌讳、一两个人名钉子 | 全员点名、剧透结局 |
| 点出跟谁的脚 / 本卷搁在何处 | 修炼简介、升级预告 |
| 真假不论、作者不站队 | 揭唯一真相 |
| 收在「雨刚停 / 面热了」一类开场钩 | 长篇设定附录 |

体量：约 **800–1500 字**（可短于正章）。文言/诗句点缀从简：半句碑文、半阙回目即可，通篇不文言。  
工程向路径表可放正文末「本卷主章路径」小注，或只进 catalog / 卷 README，**勿写进说书人嘴里**。

## 五、发布层可改 / 仓库核勿动

| 可改（发布/网页呈现） | 勿动（定锚） |
|---|---|
| 章序、章名、入卷小引 | 地脉互链核 |
| 短章合并、幕间标签 | 双读不站队、十八回阙 |
| G/R 场景化改写（另存发布稿亦可） | 无升级数值流 |
| 番茄简介、导流句 | P001 与河东不合并正文 |

若发布稿与仓库正文分叉：仓库仍是唯一设定源；发布稿差异记在卷 README 或 `创作日志`，禁止静默改核。

## 六、网页必须体现

1. 地脉组详情页：**本卷荐读序**（按 `reading_path`）优先于无序列表。  
2. 阅读器在卷上下文中：上一条/下一条走**荐读序**，不是字典序。  
3. 角标可读作：`探游 · 河东卷`（有卷名时）。  
4. 入世页露出「卷」而非仅「条数」。  

字段源：`stories/catalog.json` → `volumes` + `groups` + `entries`。
