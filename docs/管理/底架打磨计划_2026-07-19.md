## 一、目标架构（打磨后）

```text
事实源（唯一）
  docs/**                 设定圣经
  stories/<地脉>/         正文
  stories/人物志/         档案 N
  stories/世界/           势力/地标/生灵/物产/风物碎片
  stories/catalog.json    机器索引（entries + volumes + world）

工具链
  scripts/scaffold_entry.py   新条目一键建
  scripts/check_integrity.py  工程总检
  scripts/postcheck_story.py  正文词表（已有）
  scripts/check_reviews.py    审印（已有）
  npm run check               integrity + reviews

网页
  web/  只读展示；业务清单不写死在 JS
  数据：catalog.entries + catalog.world + /project/*.md
  URL：#/home | #/vol/:id | #/read/:entryId | #/world/:tab | #/seal/:stage
```

**硬规则：**

1. 正文 / 档案 / 世界碎片 **先落盘**，再挂 catalog；禁止只改网页假数据。  
2. 网页硬编码仅允许：UI 文案、PLACE_VERSE 诗句、审印阶段元数据。  
3. 改 `正文.md`：postcheck → 必要时 review 待勘 → integrity。
