# 交稿清单（一页纸）

> 新条目从脚手架到网页可见的最低路径。规范：`catalog.schema.md` · `21` 审查 · `20` 创作。

## 1. 开条

```bash
# 正文 / 人物志
python3 scripts/scaffold_entry.py \
  --id Z007 --genre Z --group hedong --volume vol-hedong --track main --title 夜不入龛

# 世界碎片
python3 scripts/scaffold_entry.py --world \
  --id WLD-XX --kind faction --name 某帮 --group hedong
```

- 主线 **不要** 指望脚手架自动写 `main_path`；确认后再手改 `catalog.json` volumes。
- 新稿 `review.state` 默认 **pending（待勘）**。

## 2. 写作

- 正文只改目录内 `正文.md`（勿改 `种子_*.md` 指针）。
- frontmatter 保持 id / genre / group / volume / track 与 catalog 一致。
- 改已过审稿：升高 `Version:`，并把 catalog `review.state` 改回 `pending`（或等 check_reviews 报失效）。

## 3. 机器自检

```bash
python3 scripts/postcheck_story.py stories/<地脉>/<编号_标题>/正文.md --strict
python3 scripts/check_integrity.py
python3 scripts/check_reviews.py
# 或
npm run check
```

退出码非 0 不得当交稿。

## 4. 基础审查

- 对照 `docs/原则/21_故事审查标准.md` **v0.4**：**S0 归属四问** + R0–R3 + 形态卡模块。
- 写 `07_审查报告.md`（模板已含 S0）或记入 `stories/审查总簿.md`；主线可填 13 第四节世界痕迹一行。
- 总编审回写 catalog `review`（state / seal / source_version / **standard_version 0.4** / date / report）。

## 5. 索引与网页

- `main_path` / `reading_path` 仅在确认连载序后更新。
- 本地：`npm start` → `http://127.0.0.1:4182/`
- 分享章链：`#/read/<id>` · 卷 `#/vol/<groupId>` · 审印 `#/seal/pending` · 世界 `#/world/factions`

## 6. 收尾

- [ ] postcheck PASS  
- [ ] integrity 无 error  
- [ ] review 字段完整（待勘可无 report）  
- [ ] 组 README / 目录骨架状态（若主线）  
- [ ] 重要决策记 `创作日志.md`（可选）
